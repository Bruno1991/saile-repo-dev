# Anti-Drift Rules

- não alterar objetivo durante implementação sem registrar decisão;
- não introduzir framework paralelo;
- não duplicar contratos existentes;
- não criar pasta genérica `utils` para esconder responsabilidades;
- não deixar documentação contradizer código deliberadamente;
- não considerar dump antigo mais confiável que repositório atual;
- não trocar ID/nome do projeto por preferência;
- não adicionar feature futura durante correção isolada.

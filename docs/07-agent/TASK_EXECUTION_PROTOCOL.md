# Task Execution Protocol

1. formular objetivo e critério de aceitação;
2. mapear módulos e contratos;
3. consultar skills;
4. reproduzir problema ou medir baseline;
5. implementar menor mudança completa;
6. validar localmente;
7. revisar diff e segredos;
8. atualizar documentos;
9. entregar evidência.

Para tarefas grandes, decompor em unidades que mantenham o repositório executável.

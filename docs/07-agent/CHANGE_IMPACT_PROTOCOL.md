# Change Impact Protocol

Avaliar impacto em:

- API/rotas/settings;
- banco e migração;
- provider e rede;
- UI e localização;
- playback;
- compatibilidade e performance;
- segurança e privacidade;
- build/release;
- documentação e testes.

Impacto “nenhum” deve ser justificado quando a área é plausivelmente afetada.

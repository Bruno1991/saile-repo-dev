# Build and Release

## Build

- iniciar em árvore limpa;
- validar Python, XML, docs e segredos;
- obter versão de fonte única;
- empacotar diretório raiz correto;
- excluir arquivos de desenvolvimento;
- gerar `addons.xml` e hashes por ferramenta;
- inspecionar ZIP e manifest;
- salvar relatório.

## Release

- atualizar changelog e status;
- executar testes de instalação e atualização;
- publicar artefatos imutáveis;
- validar URL do repositório;
- registrar commit, hashes e compatibilidade.

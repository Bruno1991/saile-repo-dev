# Versioning Policy

Usar versionamento semântico como intenção:

- PATCH: correção compatível;
- MINOR: funcionalidade compatível;
- MAJOR: quebra de contrato ou migração relevante.

Kodi/repositório podem impor formato adicional, mas a versão deve ser única e coerente. Build não deve incrementar versão silenciosamente sem registro no changelog.
